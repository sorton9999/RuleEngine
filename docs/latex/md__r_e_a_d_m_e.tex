Rule\+Engine -- An API that can create a rules-\/based, pattern matching engine using the Rete algorithm.

From Wikipedia:\+ The Rete algorithm provides a generalized logical description of an implementation of functionality responsible for matching data tuples ("{}facts"{}) against productions ("{}rules"{}) in a pattern-\/matching production system. The word \textquotesingle{}Rete\textquotesingle{} is Latin for \textquotesingle{}net\textquotesingle{} or \textquotesingle{}comb\textquotesingle{}. The same word is used in modern Italian to mean \textquotesingle{}network\textquotesingle{}.

This is a full implementation of the algorithm which provides logical AND and OR operators to rules that are input via the API. The implementation offers the operations up in a rule builder that follows the fluent software pattern.

A typical rule can be put together in the following way:\+


\begin{DoxyCode}{0}

\DoxyCodeLine{\ \ \ \ \ \ \ \ \ \ .Or<Customer>\ (\textcolor{stringliteral}{"{}CustomerNewEngland"{}},\ (token,\ customer)\ =>\ customer.State\ ==\ \textcolor{stringliteral}{"{}Rhode\ Island"{}},}
\DoxyCodeLine{\ \ \ \ \ \ \ \ \ \ \ \ \ \ (token,\ customer)\ =>\ customer.State\ ==\ \textcolor{stringliteral}{"{}Maine"{}})}

\end{DoxyCode}


Items or facts are input using:\+ {\ttfamily rule\+Engine.\+Assert(\+Customer\+List)}

The rule(s) are fired using:\+ {\ttfamily rule\+Engine.\+Fire\+All()}

Using this rule engine, it is possible to write multiple rules using the logical operators and run it against input and then provide an action to run against the output. These actions are performed in the .Then method.
